# 外部发现日报 Markdown 模板

> 用途：当用户已经确认想看的领域后，直接按这个模板生成可交付的 `.md` 报告文件。

```md
# {领域名称} 外部发现快报 - YYYY-MM-DD

## 今日关注范围
- 领域：{例如：AI agent / 开源项目 / 中国 AI 产品}
- 地域范围：{国内 / 国外 / 全球}
- 侧重点：{技术 / 产品 / 商业 / 综合}
- 时间范围：{最近 24 小时 / 最近 7 天}

## 今日最值得看的 3～5 条

### 1. {标题}
- 发生了什么：
- 为什么值得看：
- 这意味着什么：
- 建议是否继续跟踪：{是 / 否}

### 2. {标题}
- 发生了什么：
- 为什么值得看：
- 这意味着什么：
- 建议是否继续跟踪：{是 / 否}

### 3. {标题}
- 发生了什么：
- 为什么值得看：
- 这意味着什么：
- 建议是否继续跟踪：{是 / 否}

## 今日趋势总结
- 趋势 1：
- 趋势 2：
- 趋势 3：

## 值得继续跟踪的对象
- 仓库 / 项目：
- 产品 / 公司：
- 媒体 / 信息源：
- 关键词：

## 下一步建议
- 明天继续看什么：
- 哪些值得深挖：
- 哪些可以先忽略：

## 可选：原始来源
- {链接 1}
- {链接 2}
- {链接 3}
```

---

## 使用说明

1. 先确认用户想看的领域
2. 再按 workflow 去搜集和提纯信息
3. 最后把结果填进这个模板
4. 输出为 `.md` 文件交付给用户

---

## 文件名建议

```text
领域名-daily-briefing-YYYY-MM-DD.md
```

例如：

```text
ai-agent-daily-briefing-2026-03-17.md
china-ai-product-daily-briefing-2026-03-17.md
open-source-ai-daily-briefing-2026-03-17.md
```
