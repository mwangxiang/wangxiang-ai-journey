# 外部发现日报执行提示词模板

> 用途：把这段提示词直接交给 AI，让它在确认用户关注领域后，执行“搜集外部信息 → 提纯信号 → 生成 Markdown 报告”的流程。

## 提示词模板

```text
你要执行一个“外部发现日报”工作流。

目标不是泛泛聊天，而是：
在确认用户想看的领域后，搜集外部世界里值得关注的信息，进行筛选、提炼和判断，最后输出一份 Markdown 报告文档。

请严格遵守以下要求：

1. 先确认用户想看的领域
- 先弄清楚用户关注的领域是什么
- 还要确认侧重点是技术、产品、商业还是综合
- 还要确认关注范围是国内、国外还是全球
- 如果这些信息不清楚，先补齐再开始搜集

2. 搜集信息时，不要按平台堆新闻
- 不要写成“某平台有什么、某平台有什么”的流水账
- 要优先寻找真正有价值的外部信号

3. 对信息按信号类型进行提炼
优先识别以下几类：
- 爆发信号：突然大量出现、迅速升温的话题或对象
- 转向信号：说明行业风向变化的内容
- 工具信号：新工具、新仓库、新产品、新工作流壳层
- 情绪信号：大家在兴奋、吐槽、担心什么
- 机会信号：哪些变化意味着新的产品机会、内容机会或交付机会

4. 最终只保留最值得看的 3～5 条
每条必须包含：
- 发生了什么
- 为什么值得看
- 这意味着什么

5. 必须补充趋势总结
- 说明这些信号背后有没有共同方向
- 说明这些变化是分散噪音，还是一个更大趋势的不同表现

6. 最终必须输出为 Markdown 文档
- 不要只给聊天回复
- 要生成一个 `.md` 文件作为正式交付物
- 文件内容必须结构清晰，方便阅读、转发和归档

7. 建议的 Markdown 结构如下：
- 标题
- 今日关注范围
- 今日最值得看的 3～5 条
- 今日趋势总结
- 值得继续跟踪的对象
- 下一步建议
- 可选：原始来源链接

8. 输出要求
- 内容简洁，但必须有判断
- 不要堆无意义信息
- 不要只做新闻搬运
- 重点是“信号提纯 + 判断 + Markdown 文档交付”

当你完成后，返回 Markdown 文档内容，或者直接生成对应的 `.md` 文件。
```

---

## 推荐搭配方式

建议和下面两个文件一起使用：

- `external-signal-daily-briefing-method.md`
- `external-signal-daily-briefing-template.md`

这样就形成完整三件套：

1. 方法说明
2. 输出模板
3. 执行提示词
