# Daily Index and Handoff

This guide explains how I keep a long-running AI-assisted project from turning into chaos.

## Why this matters

When a project spans multiple days, three things tend to happen:

- details get lost
- unfinished work becomes hard to resume
- "memory" grows until it becomes noisy and expensive

To solve this, I separate information into layers instead of dumping everything into one place.

## The 4-layer structure

### 1. Content files
These hold the real substance:
- raw notes
- extracted text
- refined text
- design drafts
- conversation drafts
- process artifacts

### 2. Handoff documents
These exist for unfinished work.
Each unfinished project should have a handoff file that records:
- current progress
- what is done
- what is not done
- blockers
- next actions
- related files

### 3. Daily index
The daily note is not a full log dump.
It acts as a navigation page for the day:
- what happened today
- key decisions
- file indexes
- unfinished-project handoff indexes
- next actions

### 4. Long-term memory
Long-term memory should only keep stable, high-value information:
- durable rules
- durable preferences
- stable workflows
- important lessons

## Core rule

**Content lives in files. Daily notes point to files. Long-term memory stores only what should survive long-term.**

## Example daily note sections

- Today summary
- Key decisions
- File index
- Unfinished project handoff index
- Next actions

## Example handoff sections

- Current progress
- Completed
- Not completed
- Current blockers
- Next actions
- Related file index
- Notes

## Why I like this approach

It keeps the system:
- resumable
- searchable
- lightweight
- suitable for public sharing after desensitization

This structure is simple enough to maintain, but strong enough to support longer-term work.
