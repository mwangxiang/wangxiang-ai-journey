# Guide：第一次看这个项目，应该怎么读

如果你第一次来到这个仓库，最容易出现的情况是：

- 不知道从哪开始
- 不知道哪些是日志，哪些是教程
- 不知道这个项目到底是在展示什么

所以可以按下面这个顺序看。

## 第一步：先看 README
先理解这个项目的目标、定位、原则，以及它为什么存在。

## 第二步：看 ROADMAP
README 解决“这是什么”，ROADMAP 解决“现在做到哪了、后面要去哪”。

## 第三步：看 Learning Log
如果你想看这个项目是怎么一步一步搭起来的，优先看：
- `docs/learning-log/`

## 第四步：看 Case Studies
如果你更关心完整问题是怎么解决的，优先看：
- `docs/case-studies/`

## 第五步：看 Workflows
如果你想直接复制某种方法，优先看：
- `docs/workflows/`

## 第六步：看 Guides
如果你完全是新手，或者想找更面向入门的内容，再看：
- `docs/guides/`

## 最后提醒
这个项目不是“已经彻底完成的成品”，而是一个正在公开成长中的项目。

所以你看到的，不只是结果，也包括中间过程、调整、取舍和不完美。
