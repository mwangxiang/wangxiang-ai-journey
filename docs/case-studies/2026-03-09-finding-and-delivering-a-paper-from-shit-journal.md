# 案例复盘：如何从 S.H.I.T Journal 精准找到目标论文，并把文件直接发给用户

## 任务目标

用户想找的是一篇具体论文：

**《地府货币膨胀：东亚父母该烧多少钱才能保证孩子不会乱花》**

用户的真实需求不是“帮我随便看看这个网站”，而是：
- 尽可能准确地找到目标论文
- 最终把文件直接发到聊天里
- 不要只给一个可能打不开的外链

这意味着，任务成功的标准不是“知道大概是什么”，而是：
**准确命中 + 成功送达。**

---

## 一开始的问题

最开始只有一个名字：
- `S.H.I.T`

这会带来两个问题：

### 1. 搜索噪音很大
这个名字本身太像普通词汇，直接全网搜时非常容易被噪音淹没。

### 2. 工具并不稳定
过程中出现了：
- 搜索接口不稳定
- 浏览器工具不可用
- 网页内容前端渲染

如果继续只靠“普通搜索 + 打开网页点点点”，效率会很差。

---

## 关键转折

### 第一步：先确认官网
最后确认到官网：
- `https://shitjournal.org/`

### 第二步：从前端代码里找真实数据入口
因为首页是前端渲染，所以改为抓取前端 JS bundle，进而提取出：
- API 域名：`https://api.shitjournal.org`

### 第三步：发现它公开了 OpenAPI 文档
继续检查后发现：
- `https://api.shitjournal.org/openapi.json`
- `https://api.shitjournal.org/docs`

这一步极大降低了猜接口的成本。

### 第四步：使用站内搜索，而不是继续依赖外部搜索
最终真正命中的查询是：
- `/api/search/article?q=地府货币膨胀`

命中结果为：
- **《地府货币膨胀：东亚父母该烧多少钱才能保证孩子不会乱花》**

### 第五步：继续取单篇详情和 PDF 地址
再通过：
- `/api/articles/{article_id}`

拿到：
- 文章详情
- 作者
- 评分
- PDF 直链

### 第六步：下载到本地后再发给用户
考虑到用户前面已经反馈过：
- 某些外链/文件在聊天环境里不一定能直接打开

所以最终没有只发 PDF URL，
而是：
1. 下载到本地临时文件
2. 直接通过聊天发送文件

---

## 最终结果

成功找到并交付：

**《地府货币膨胀：东亚父母该烧多少钱才能保证孩子不会乱花》**

并额外确认到：
- 作者：`wabn`
- 分类：`meme`
- 分区：`stone`
- 评分：3477 人评分，平均 4.90/5

---

## 这次任务最值得记住的点

### 1. 先找系统，再找目标
如果目标位于某个特定站点内部，先摸清站点的数据结构，通常比无脑搜标题更快。

### 2. 浏览器不是唯一工作方式
浏览器不可用时，仍然可以通过：
- 前端 bundle 分析
- OpenAPI 文档
- 站内 JSON API
来完成任务。

### 3. 命中结果不等于任务完成
真正完成任务的是：
- 找到内容
- 拿到文件
- 发给用户

### 4. 一次成功应该沉淀成 workflow
这类任务以后还会反复出现。
只有把这次过程整理成标准 workflow，才能真正变成长期资产。

因为它很能体现 OpenClaw 在真实工作流中的价值：
- 不是只会“搜索”
- 而是会“定位系统 → 找接口 → 命中目标 → 交付文件”
