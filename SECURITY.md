# SECURITY

这个项目以公开记录为主，不是高风险生产系统，但依然遵守基本安全原则。

## 不应公开的内容

以下内容不应进入仓库：

- API Key
- Token
- 密码
- Cookie / Session
- 私人邮箱、手机号、住址等个人敏感信息
- 服务器 IP、内网结构、机器登录方式
- 任何可直接用于接管账号或基础设施的信息

## 发现问题怎么办

如果你发现仓库中存在：
- 未脱敏的敏感信息
- 可能导致安全风险的配置片段
- 不适合公开的真实记录

请不要扩散，优先通过私下方式提醒维护者处理。

## 当前策略

- 默认先脱敏，再公开
- 无法确定是否敏感时，宁可先不公开
- 对历史内容做定期复查
